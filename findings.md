# Findings

## 需求阶段结论

- 该产品不应被定义为“纯 API 自动化工具”，而应定义为“浏览器执行优先的动作系统”。
- 网络录制的核心价值在于帮助系统和人工理解用户行为、字段来源、依赖关系和参数语义。
- 人工审核不是补充流程，而是首版中的主流程之一。
- 若首版直接追求结构化业务动作自动生成，风险过高；应先让系统稳定沉淀可回放宏，再逐步抽象为业务动作。
- 对用户而言，登录态复用和执行稳定性比“完全脱离浏览器”更重要。
- 首版证据范围进一步收口为“网络请求/响应 + 页面导航 + 会话状态 + 文件上传下载”，细粒度 `DOM` 事件轨迹不再作为 MVP 必选项。
- 技术栈已正式从 `Java` 路线切换为 `Python 3.11+ + FastAPI + React + Playwright Python + SQLite`。
- 长任务通信方案已收口为 `SSE`，覆盖录制状态、元数据分析和动作执行日志。
- `MetadataDraft` 统一定义为录制结束后异步分析生成，而不是录制链路的直接同步产物。
- 主文档已按“产品文档”和“技术文档”分类迁移，相关引用统一指向新路径。
- `docs/技术文档/开发步骤拆解.md` 已被补强为可直接查看当前阶段状态的进度看板。
- 仓库开发前置已完成：本地 `git`、GitHub 远程、基础忽略规则、项目说明和环境样例文件均已就绪。
