## 当前已确认的关键决策

- 产品首版形态采用 `本地服务 + Web 管理台 + 受控浏览器`。
- 执行层以 `浏览器重放` 为主，接口记录作为分析和抽象动作的证据。
- 浏览器会话首版使用工具维护的独立 `Profile`，后续再考虑兼容用户本机已有 `Profile`。
- 录制范围首版以网络请求响应、URL/跳转、Cookie/Storage、文件上传下载和页面阶段信息为主，细粒度 `DOM` 事件轨迹降级为后续增强项。
- 首版需要完整的管理台支持人工审核。
- 动作抽象采用渐进式模型：先形成可回放宏，再提升为结构化业务动作。
- 首版支持导出/导入，暂不要求多人协作。
- 技术路线正式采用 `Python 3.11+ + FastAPI + React + Playwright Python + SQLite`。
- 开发期采用前后端分离，运行期采用后端挂载前端产物的本地一体化模式。
- 长任务状态推送统一采用 `SSE`。
- 文档结构按“产品文档 / 技术文档”两个分类目录组织，主文档已迁移到对应目录中。
- 当前仓库已完成本地 `git` 初始化、远程绑定和首个开发前置提交准备。
