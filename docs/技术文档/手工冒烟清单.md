# 手工冒烟清单

本文档用于阶段 7 的半自动收尾验证。建议先跑自动化验证，再按本文完成一次人工浏览器走查。

下文命令默认都在**仓库根目录**执行。

## 0. 先跑自动化验证

在仓库根目录执行：

```bash
cd backend && ../.venv/bin/python -m pytest tests -v
cd frontend && npm run test:run
cd frontend && npm run build
```

只有自动化验证全绿，再进入人工走查。

## 1. 冒烟前准备

- 选择一种启动方式：
  - 开发模式：`./scripts/dev/start_backend.sh` + `./scripts/dev/start_frontend.sh`
  - 运行模式：`./scripts/run/start_local_app.sh`
- 默认推荐使用公开站点：`https://httpbin.org/forms/post`
  - 推荐手工动作：填写 `custname`，选择一个 `size`，勾选至少一个 `topping`，然后点击 `Submit order`
  - 这条链路会产生一次明确的 `POST` 请求，并跳转到回显页，适合录制 / 审核 / 生成宏 / 执行闭环
  - 如果当前网络环境无法访问该站点，再替换为满足同等条件的内部测试表单页
- 若要验证“导入成功”，建议准备第二份干净数据目录，例如：

```bash
WEBTOACTIONS_DATA_DIR=.webtoactions-import APP_PORT=8010 ./scripts/run/start_local_app.sh
```

## 2. 录制 -> 审核 -> 生成宏 -> 执行

- [ ] 打开管理台首页，确认能访问 `/api/health`
- [ ] 进入“会话管理”，创建一个新浏览器会话
- [ ] 进入“新建录制”，填写 `https://httpbin.org/forms/post`，开始录制
- [ ] 在受控浏览器中填写 `custname`，选择一个 `size`，勾选至少一个 `topping`，然后点击 `Submit order`
- [ ] 返回管理台，结束录制
- [ ] 打开录制详情，确认能看到页面阶段、请求记录、会话快照或文件传输摘要
- [ ] 进入审核页，等待分析完成
- [ ] 标注至少一个关键请求，填写字段说明 / 参数来源后保存审核结果
- [ ] 点击“生成动作宏”，跳转到动作详情
- [ ] 在动作详情填写参数，选择浏览器会话并发起执行
- [ ] 打开执行详情，确认能看到执行状态、步骤日志和最终页面或失败原因

## 3. 导出 -> 导入

- [ ] 进入“导入导出”页，选择刚完成的录制并导出资料包
- [ ] 记录导出的 zip 文件名与导出时的录制 ID
- [ ] 切到一份干净数据目录启动的实例，或清空后重新启动的实例
- [ ] 在“导入导出”页上传刚导出的资料包
- [ ] 确认页面提示导入成功，并显示“不迁移登录态 / 继续执行需准备新的可用会话”之类的警告
- [ ] 回到录制列表、动作库、执行中心，确认导入的录制、动作宏、执行记录都可见

## 4. 运行模式额外检查

- [ ] 使用单进程模式启动后，直接访问根路径 `/` 正常返回前端页面
- [ ] 直接访问深层路由（如 `/importexport` 或 `/actions`）不会 404
- [ ] `/api/health` 仍然返回 JSON，而不是前端页面

## 5. 通过标准

本次手工冒烟可视为通过，需要同时满足：

1. 录制、审核、动作宏、执行、导入导出五段链路都能走通
2. 导入后相关记录可重新查看
3. 运行模式下根路径、深层路由与 `/api/health` 同时正常
4. 过程中没有出现无法定位的错误或静默失败
