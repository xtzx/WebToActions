# 本地启动说明

本文档说明 `WebToActions` 当前支持的两种本地运行方式：

- 开发模式：前后端分离，适合日常开发与调试
- 运行模式：单进程一体化，适合阶段 6/7 验证与演示

## 前置条件

- 仓库根目录存在 `.venv`，且其中安装了后端依赖
- 已在 `frontend/` 安装前端依赖
- 本机可用 `Python 3.11+`
- 下文命令默认都在**仓库根目录**执行

## 首次初始化

如果你是第一次拉起仓库，可先执行：

```bash
python3.11 -m venv .venv
./.venv/bin/pip install -e "./backend[dev]"
./.venv/bin/playwright install chromium
cd frontend && npm install
```

如果 `.venv` 与 `frontend/node_modules` 已经存在，可跳过本节。

## 开发模式

后端：

```bash
./scripts/dev/start_backend.sh
```

前端：

```bash
./scripts/dev/start_frontend.sh
```

访问地址：

- 前端管理台：`http://127.0.0.1:5173`
- 后端健康检查：`http://127.0.0.1:8000/api/health`

如果 `5173` 端口被占用，请查看前端启动终端输出中的实际地址；必要时先释放端口后重试。

适用场景：

- 前端页面调试
- 后端接口开发
- 录制 / 审核 / 动作 / 执行链路联调

## 运行模式

单进程运行会先构建前端，再由后端直接挂载 `frontend/dist`：

```bash
./scripts/run/start_local_app.sh
```

默认地址：

- 管理台：`http://127.0.0.1:8000`
- 健康检查：`http://127.0.0.1:8000/api/health`

如果 `8000` 端口被占用，可覆盖端口：

```bash
APP_PORT=8010 ./scripts/run/start_local_app.sh
```

如果需要一份干净数据目录做导入验证，可同时覆盖数据目录：

```bash
WEBTOACTIONS_DATA_DIR=.webtoactions-import APP_PORT=8010 ./scripts/run/start_local_app.sh
```

## 推荐验证命令

后端：

```bash
cd backend && ../.venv/bin/python -m pytest tests -v
```

前端测试：

```bash
cd frontend && npm run test:run
```

前端构建：

```bash
cd frontend && npm run build
```

## 最低验证清单

开发模式或运行模式启动后，至少确认以下项目：

1. 首页可访问
2. `/api/health` 返回 `status=ok`
3. 深层路由（如 `/importexport`）直接访问不会 404
4. 录制列表、审核页、动作库、执行中心、导入导出页可正常打开
